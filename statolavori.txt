VALUE4CARE PIPELINE
Stato del progetto e piano operativo

---

1. OBIETTIVO GENERALE

---

Costruire una pipeline completamente offline e privacy first per trasformare fascicoli sanitari in event log strutturati utilizzabili per process mining.

Flusso logico finale desiderato:

PDF fascicolo sanitario
-> estrazione testo
-> estrazione strutturata tramite LLM locale
-> validazione backend
-> salvataggio in database relazionale
-> generazione event log
-> process mining
-> visualizzazione paziente singolo
-> visualizzazione per malattia o coorte

Vincoli fondamentali:

* Nessuna chiamata esterna consentita
* LLM esclusivamente locale
* Separazione netta tra LLM e database
* Architettura scalabile a milioni di pazienti
* Privacy massima

Modello target scelto:
qwen3:30b-a3b tramite Ollama
